## 转账 UI/数据流重构：方案 B（转账仅搜索，不进交易流）
- 用户判定：把 transfers 套进 transactions 流（合并 + 复用通用 TransactionItem/详情）是错的，转账与交易本质不同（双账户/双币种/汇率/手续费）。
- 决策（用户选 B）：转账不进首页最近交易 / 交易列表 / 日历 / 统计，只在搜索里用专属 UI 单独处理。
- 实现：
  - `AppContext.tsx`：`loadMergedTransactions` 拆为 `loadTransactions`(仅支出/收入) + `loadAllTransfers`(新 transfers 表 + 历史 transactions 表 to_account_id 记录，统一为独立 `transfers` 流)；`mapTransaction` 不再含转账分支；新增 `mapTransferFromTxn`；`calculateAccountBalances` 改为接收独立 `transfers` 按 from/to 双账户按币种算余额；`addTransaction/deleteTransaction/updateTransaction` transfer 分支改走 `setTransfers`；context 导出 `transfers`。
  - `Home/TransactionList/Calendar`：移除转账分支（流中已无转账）；移除 `formatTransferAmount` 导入。
  - 新建 `components/ui/TransferItem.tsx`：专属转账卡片（出→入账户、原/目标币种金额、手续费、汇率、日期）。
  - `Search.tsx`：接入 `transfers`；`type=转账` 时用 `filteredTransfers` + `TransferItem` 专属渲染；详情复用 Search 内置的转账感知 BottomSheet。
- 验证：`vite build` 通过（7.72s）。AppContext/Search/Home 等 lint 仅剩项目既有错误（toYMD 缺失、big_expense_threshold、deleteBill 缺失），本次未引入新错误。
- 已知限制（聚焦 B，未做）：① 搜索账户/关键字筛选转账时仅匹配"出账户/原币种"（applyFilters 只看 accountId）；② 搜索"最近交易"无查询态不显示转账；③ 转账账户改名后列表 accountName 不实时刷新（需 reload）。这些属方案 A 范畴，排期再做。

## 转账逻辑重构（transfers 独立表 + 多币种）
- 用户要求：转账不计入消费，应从 transactions 移到独立 transfers 表；transfers 表支持多币种。
- 决策（用户确认）：**完整多币种**；**不迁移历史**（旧 to_account_id 行保留在 transactions，仍由 `to_account_id` 检测映射为 `type='transfer'`）。
- 改动：
  - `supabase/migrations/017_transfers_standalone_multicurrency.sql`：DROP 旧 transfers（transactions 扩展表，从未写入），重建为独立表 `from_currency/from_amount/to_currency/to_amount/exchange_rate/fee`；RLS policy 重建；`get_sync_counts` 增加 transfers 计数。
  - 本地层：`db/database.ts` 升 v10 加 `transfers` store + `TransferRecord`；`db/sync-engine.ts` TABLE_NAMES/MAP/clearAllData 加 transfers；`db/local-operations.ts` 加 `localTransfers`（沿用 newRecordBase/markDirty/markDeleted）。
  - `AppContext`：`loadMergedTransactions` 合并 transactions+transfers；`addTransaction` 转账走 `localTransfers`；`updateTransaction`/`deleteTransaction` 按 `source('txn'|'transfer')` 路由；余额按 `fromAmount`/`toAmount`（各自币种）计算。
  - `AddTransaction`：多币种转账输入（手续费、跨币种到账金额）；`TransactionItem`/`TransactionList`/`Search`/`Home`/`TransactionDetailSheet` 用 `formatTransferAmount` 展示。
